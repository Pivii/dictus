<task type="auto">
  <name>Task 2: Fix cross-process transcription display</name>
  <files>DictusKeyboard/KeyboardRootView.swift, DictusKeyboard/KeyboardState.swift, DictusApp/DictationCoordinator.swift</files>
  <action>
  THREE changes required:

  **A. Consolidate Darwin notifications in DictationCoordinator (DictationCoordinator.swift):**

  In `startDictation()`, the current code calls `writeTranscription(stubResult)` (which posts transcriptionReady) then `updateStatus(.ready)` (which posts statusChanged). This creates a race condition because two Darwin notifications fire sequentially and the keyboard may read UserDefaults between them.

  Fix: In the completion block of `startDictation()` (after the transcription sleep), inline the final writes instead of calling `writeTranscription` + `updateStatus` separately:
  1. Write lastTranscription and lastTranscriptionTimestamp to UserDefaults
  2. Write .ready status to UserDefaults
  3. Set local published properties (lastResult, status)
  4. Call `defaults.synchronize()` once
  5. Post `statusChanged` notification
  6. Post `transcriptionReady` notification

  Keep the existing `updateStatus()` method unchanged for other status transitions (.recording, .transcribing, .idle, .failed). Keep `writeTranscription()` as a helper but remove the Darwin notification post from inside it, OR simply inline the writes.

  **B. Make refreshFromDefaults() also read lastTranscription (KeyboardState.swift):**

  In `refreshFromDefaults()`, after reading `dictationStatus`, also read `lastTranscription` when status is `.ready`:
  ```swift
  if status == .ready {
      lastTranscription = defaults.string(forKey: SharedKeys.lastTranscription)
  }
  ```

  This ensures that even if only the `statusChanged` notification fires, the keyboard picks up the transcription text when status is `.ready`. Belt-and-suspenders fix alongside the notification consolidation.

  **C. Make StatusBar spinner conditional (KeyboardRootView.swift):**

  Change `StatusBar` to accept a `showSpinner: Bool` parameter (default `true`). Only render `ProgressView()` when `showSpinner` is true.

  Update the StatusBar call site in `KeyboardRootView.body`:
  ```swift
  if let message = state.statusMessage {
      StatusBar(
          message: message,
          showSpinner: state.dictationStatus != .ready && state.dictationStatus != .failed
      )
  }
  ```
