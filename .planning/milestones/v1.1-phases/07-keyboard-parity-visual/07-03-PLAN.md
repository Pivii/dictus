1. **Rework BrandWaveform to use Canvas rendering** — Replace the `ForEach + RoundedRectangle` approach in `waveformContent` with `Canvas`:
   ```swift
   private func waveformContent(processingPhase: Double) -> some View {
       Canvas { context, size in
           let totalSpacing = barSpacing * CGFloat(barCount - 1)
           let barWidth = max((size.width - totalSpacing) / CGFloat(barCount), 2)

           for index in 0..<barCount {
               let energy = energyForBar(at: index, processingPhase: processingPhase)
               // Zero minHeight when NOT processing — bars disappear at zero energy
               let minHeight: CGFloat = isProcessing ? 4 : 0
               let height = max(minHeight + CGFloat(energy) * (maxHeight - minHeight), isProcessing ? 4 : 0)

               let x = CGFloat(index) * (barWidth + barSpacing)
               let y = (size.height - height) / 2
               let rect = CGRect(x: x, y: y, width: barWidth, height: max(height, 0))
               let path = Path(roundedRect: rect, cornerRadius: barWidth / 2)

               context.fill(path, with: .color(resolvedBarColor(at: index)))
           }
       }
       .frame(height: maxHeight)
   }
   ```
   - **Fix zero-energy stillness:** When `isProcessing == false` and energy is 0, the bar height should be exactly 0 (currently `minHeight: CGFloat = 4` causes micro-movements). Set `minHeight = 0` for non-processing mode.
   - **Color resolution for Canvas:** Canvas cannot use `some ShapeStyle` — it needs resolved `Color` values. Create a new method `resolvedBarColor(at:) -> Color` that returns a plain Color (not `AnyShapeStyle`). For the center gradient bars, pick the midpoint color `Color.dictusGradientStart` (Canvas doesn't support gradients per-path easily — use a solid brand blue for center bars). This is a minor visual simplification acceptable for Canvas performance.
   - **Keep TimelineView** wrapping Canvas for processing mode animation — the existing `TimelineView(.animation)` pattern works perfectly.
   - **Keep the `.animation(.easeOut(duration: 0.08), value: energyLevels)` on the non-processing branch** — Canvas will still animate smoothly via SwiftUI's animation system.
   - Remove the old `barView()` method and the `colorForBar()` returning `some ShapeStyle`.
