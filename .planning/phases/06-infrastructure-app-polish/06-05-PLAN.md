**Fix 2 — Sinusoidal processing waveform (BrandWaveform.swift + RecordingView.swift):**

In BrandWaveform.swift, add a processing mode that generates synthetic sinusoidal energy levels:

Add a new public property `isProcessing: Bool = false` to BrandWaveform. When true, ignore the `energyLevels` parameter and instead generate a smooth sinusoidal wave pattern using a TimelineView or timer-driven @State:

```swift
public struct BrandWaveform: View {
    public let energyLevels: [Float]
    public var maxHeight: CGFloat = 80
    public var isProcessing: Bool = false

    @State private var processingPhase: Double = 0
```

When `isProcessing` is true:
- Use a TimelineView(.animation) or withAnimation(.linear(duration: 2).repeatForever(autoreverses: false)) to continuously increment processingPhase
- Generate synthetic energy levels: for each bar index i, compute `energy = 0.3 + 0.4 * sin(2 * .pi * (Double(i) / Double(barCount) + processingPhase))`. This creates a smooth traveling sine wave.
- Apply lower opacity (0.4) for the overall waveform to give a subtle ambient feel during processing

Add an `.onAppear` that starts the animation when isProcessing is true:
```swift
.onAppear {
    if isProcessing {
        withAnimation(.linear(duration: 2).repeatForever(autoreverses: false)) {
            processingPhase = 1.0
        }
    }
}
```

Update the init to include isProcessing:
```swift
public init(energyLevels: [Float] = [], maxHeight: CGFloat = 80, isProcessing: Bool = false)
```
